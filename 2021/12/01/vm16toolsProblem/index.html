解决VM16安装Tools不成功的问题

问题点

将VMware WorkStation Pro 更新到16以后 再新建的虚拟机[Winodows 2008 Server R2]

安装Tools出现了Windows不信任该驱动程序的情况并要求手动安装

报出诸如下错误

安装程序无法自动安装 Virtual Machine Communication Interface Sockets (VSock)

分析原因

VMware的esxi版本6.5以上安装不是最新版的操作系统需要安装补丁才能安装vmwaretools。

针对Windows Server 2019之前的 需要打补丁，安装最新版的操作系统例如Windows Server 2019不需要。
打了补丁之后才能安装vmtools或者vmrc工具。

解决方法

安装补丁

Windows 2008 server R2

KB4490628

代号为KB4490628的补丁是微软于2019年面向Win7系统发布的服务堆栈更新，很多用户在使用操作系统、安装驱动时可能会用到，如部分独显驱动无法安装，安装了KB4490628更新之后就可以正常安装驱动

文件名称：windows6.1-kb4490628-x64.msu
文件大小：9.1MB
MD5：C4D6FB715CB1DCCBD287F03E272AC966
版本：64bit版

下载

文件名称：windows6.1-kb4490628-x86.msu
文件大小：4MB
MD5：0698531DF562EBE22591F9E73FB5F0E2

下载

KB4474419

kb4474419补丁是一款专门为微软系统打造的升级补丁

文件名称：kb4474419.msu
文件大小：53.3MB
MD5：DE218F5A7004694D2824D5505E7D2C06

下载
